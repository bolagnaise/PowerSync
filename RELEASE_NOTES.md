## What's Changed

**No Idle settings now explain Charge By Time deadline holds**

Clarified the Smart Optimization setting description to distinguish ordinary idle holds, which No Idle replaces with self-consumption, from holds required to meet an active Charge By Time target, which are deliberately preserved. This makes the configuration match the action plan and documented behavior without changing optimizer execution.
